# CONTRIBUTING

## Overview
This guide outlines how to contribute to the Glass Atrium framework, ensuring high coding standards, a robust review process, and adherence to our code of conduct.

## Coding Standards
- Follow [PEP 8](https://peps.python.org/pep-0008/) for Python style.
- All code must include docstrings and type hints.
- Write comprehensive unit tests covering all major functionality.

## Citation Standards
All outputs must include proper citations. Use the format below for each source:

```markdown
- [Author, Year], *Title*, [URL]
```

When submitting pull requests, include a reference in the PR description.

## Review Process
1. Open an issue or submit a PR.
2. A maintainer reviews the code and requests changes if needed.
3. The contributor implements requested changes and updates tests.
4. Final approval by at least two senior contributors.

## Ethical Guidelines
Every contribution must consider privacy, bias mitigation, and transparency. See `ethical_guidelines.md` for details.

## Contributing Workflow
- **For questions**: See `CONTRIBUTING.md` for examples.
- **For adding features**: Follow the contribution checklist.
- **For resolving issues**: Report using the template.

## Roles & Responsibilities
- **Maintainers**: Oversee the review process and final approvals.
- **Contributors**: Follow the citation guide and submit pull requests following the checklist.

## Documentation
All documentation must be clear, well‑structured, and include examples. Refer to the [Documentation Guide](LICENSE) for licensing details.

## Ethical Review
Contributions must pass an automated ethical validation before merging. See `ethics/` directory for criteria.

## Maintainers
All maintainers must sign off on major changes and ensure compliance with the ethical guidelines.
